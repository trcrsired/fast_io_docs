Ch11.1: File I/O

Overview

fast_io provides high-performance file I/O through several buffered file types. These wrap the native file handle with user-space buffering for maximum throughput.

1. File Types

Type Direction Description
::fast_io::ibuf_file Input only Buffered input file. Use for reading.
::fast_io::obuf_file Output only Buffered output file. Use for writing.
::fast_io::iobuf_file Input/Output Buffered bidirectional file. Use for both reading and writing.
::fast_io::u8iobuf_file Input/Output Same as iobuf_file but for UTF-8 (char8_t).

All file types are RAII-managed: the file is automatically closed when the object goes out of scope.

2. Hello World Example

Here is a complete file I/O example: 


#include <fast_io.h>
#include <fast_io_device.h>

int main() {
    using namespace ::fast_io::iomnp;

    // Write to a file
    {
        ::fast_io::obuf_file obf{"hello.txt"};
        print(obf, "Hello, File I/O!\n"
                   "Line 2: ", 42, " + ", 100, " = ", 142);
    } // file closed automatically here

    // Read from the file line by line
    {
        ::fast_io::ibuf_file ibf{"hello.txt"};
        for (::fast_io::string line; scan<true>(ibf, str_line_get(line)); ) {
            println(line);
        }
    }
}

3. Opening Modes

Use ::fast_io::open_mode to control how files are opened:

Flag Description
open_mode::in Open for reading.
open_mode::out Open for writing.
open_mode::app Append mode (writes go to end).
open_mode::trunc Truncate existing file to zero length.
open_mode::binary Binary mode (no text translation).

Combine flags with |: 


// Open for writing, create if not exists, truncate if exists
::fast_io::obuf_file out{"data.bin", ::fast_io::open_mode::out | ::fast_io::open_mode::trunc};

// Open for reading and writing
::fast_io::iobuf_file iof{"data.txt", ::fast_io::open_mode::in | ::fast_io::open_mode::out};

4. UTF-8 File I/O

For UTF-8 text files, use the u8 variants: 


#include <fast_io.h>
#include <fast_io_device.h>

int main() {
    using namespace ::fast_io::iomnp;

    ::fast_io::u8obuf_file u8obf{"utf8.txt"};
    print(u8obf, u8"Unicode: 你好世界 🌍\n");

    // Read UTF-8 file line by line
    {
        ::fast_io::u8ibuf_file u8ibf{"utf8.txt"};
        for (::fast_io::u8string line; scan<true>(u8ibf, str_line_get(line)); ) {
            println(line);
        }
    }
}

Key takeaways

  • ibuf_file for reading, obuf_file for writing, iobuf_file for both.
  • u8ibuf_file, u8obuf_file, u8iobuf_file for UTF-8 text files.
  • Files are RAII-managed — automatically closed on scope exit.
  • Use open_mode flags to control opening behavior.
  • Use using namespace ::fast_io::iomnp; (which includes ::fast_io::io) then call println / print / scan and manipulators like str_line_get directly.
  • Use scan with the str_line_get manipulator for line-by-line reading: for (::fast_io::string line; scan<true>(in, str_line_get(line)); ) { ... }
