Ch11.7: Random Access I/O

Seek and Tell

fast_io provides ::fast_io::seek() and ::fast_io::tell() to move the file position and query the current position. These work on any file type that supports random access, including obuf_file, ibuf_file, iobuf_file, and their unbuffered counterparts.

::fast_io::seek() takes a file reference and a seek operation. The seek operation can be constructed using ::fast_io::seek_set(offset), ::fast_io::seek_cur(offset), or ::fast_io::seek_end(offset), analogous to SEEK_SET, SEEK_CUR, and SEEK_END in the C standard library. 


#include <fast_io.h>
#include <fast_io_device.h>

int main()
{
    using namespace ::fast_io::iomnp;
    ::fast_io::obuf_file obf(u8"seekable.txt", ::fast_io::open_mode::out);

    // Write some data
    println(obf, "Hello, World!");

    // Get current position
    auto pos = ::fast_io::tell(obf);
    println("Current position: ", pos);

    // Seek back to the beginning
    ::fast_io::seek(obf, ::fast_io::seek_set(0));

    // Verify position
    auto new_pos = ::fast_io::tell(obf);
    println("After seek_set(0): ", new_pos);
}

Positional I/O (pread/pwrite)

Positional I/O lets you read from or write to a specific offset in a file without changing the file's current position. This is the equivalent of POSIX pread(2) and pwrite(2).

In fast_io, this is done with ::fast_io::pwrite() and ::fast_io::pread(). These functions take an additional offset argument. The file position remains unchanged after the call, which makes positional I/O inherently thread-safe for concurrent writes to different offsets of the same file.

Using pwrite for Random Writes


#include <fast_io.h>
#include <fast_io_device.h>

int main()
{
    using namespace ::fast_io::iomnp;
    ::fast_io::obuf_file obf(u8"positional_write.txt", ::fast_io::open_mode::out);

    // Reserve space by seeking to a known size
    ::fast_io::seek(obf, ::fast_io::seek_set(256zu));
    print(obf, "");

    // Write at specific offsets without disturbing file position
    ::fast_io::pwrite(obf, "AAAA", ::fast_io::seek_set(0zu));
    ::fast_io::pwrite(obf, "BBBB", ::fast_io::seek_set(100zu));
    ::fast_io::pwrite(obf, "CCCC", ::fast_io::seek_set(200zu));
}

After this program runs, the file contains "AAAA" at offset 0, "BBBB" at offset 100, and "CCCC" at offset 200. The file position is unaffected by the pwrite calls.

Using pread for Random Reads


#include <fast_io.h>
#include <fast_io_device.h>

int main()
{
    using namespace ::fast_io::iomnp;
    ::fast_io::ibuf_file ibf(u8"positional_write.txt");

    char buf1[5]{};
    char buf2[5]{};

    // Read from specific offsets
    ::fast_io::pread(ibf, ::fast_io::as_bytes(::fast_io::span{buf1, 4zu}), ::fast_io::seek_set(0zu));
    ::fast_io::pread(ibf, ::fast_io::as_bytes(::fast_io::span{buf2, 4zu}), ::fast_io::seek_set(100zu));

    // buf1 contains "AAAA", buf2 contains "BBBB"
    println("At offset 0:   ", ::fast_io::mnp::os_c_str(buf1));
    println("At offset 100: ", ::fast_io::mnp::os_c_str(buf2));
}

Thread Safety of Positional I/O

Because pread and pwrite do not modify the file position, they are safe to call from multiple threads on the same file handle simultaneously, as long as each thread accesses a non-overlapping region. This makes positional I/O an excellent choice for parallel file processing. 


#include <fast_io.h>
#include <fast_io_device.h>
#include <thread>

int main()
{
    ::fast_io::iobuf_file iobf(u8"parallel.bin",
        ::fast_io::open_mode::out | ::fast_io::open_mode::in);

    constexpr ::std::size_t chunk_size{4096zu};
    constexpr ::std::size_t thread_count{4zu};

    auto worker = [&](::std::size_t id)
    {
        // Each thread writes to its own region
        ::std::byte data[chunk_size];
        // ... fill data ...
        ::fast_io::pwrite(iobf,
            ::fast_io::as_bytes(::fast_io::span{data, chunk_size}),
            ::fast_io::seek_set(id * chunk_size));
    };

    ::std::thread threads[thread_count];
    for (::std::size_t i{}; i < thread_count; ++i)
    {
        threads[i] = ::std::thread(worker, i);
    }
    for (auto& t : threads)
    {
        t.join();
    }
}

Each thread writes to a distinct region of the file using pwrite with a different offset, so no locking is required.

Summary

  • ::fast_io::seek(file, op) — move the file position.
  • ::fast_io::tell(file) — get the current file position.
  • ::fast_io::seek_set(offset), ::fast_io::seek_cur(offset), ::fast_io::seek_end(offset) — seek operation constructors.
  • ::fast_io::pwrite(file, data, op) — write at offset without changing position.
  • ::fast_io::pread(file, buffer, op) — read from offset without changing position.
