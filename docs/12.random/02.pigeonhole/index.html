Ch12.2: The Pigeonhole Principle and Modulo Bias

Why Modulo is Wrong

When generating random numbers in a specific range, never use modulo operations like value % 100. This creates biased, non-uniform distributions due to the pigeonhole principle.

Here's why: if you have a 32-bit random value in the range [0, 232-1] and you compute value % 100, you're trying to map 4,294,967,296 possible values into 100 buckets. But 232 = 42,949,672 × 100 + 96, which means:

  • Values 0-95 will each appear 42,949,673 times
  • Values 96-99 will each appear 42,949,672 times

This creates a subtle bias: some outcomes are slightly more likely than others. While the difference seems small (about 0.0000002%), it violates the fundamental requirement of uniform randomness and can cause problems in simulations, games, and statistical applications.

The Correct Solution

Always use ::std::uniform_int_distribution from <random>. It uses rejection sampling to ensure truly uniform distribution, regardless of the range you specify.

The distribution engine handles the complexity of ensuring uniform distribution. Never try to manually convert random bytes to a range using modulo or division — you will introduce bias.

Example: Generating a Random Number in a Range (Correct Way)


#include <fast_io.h>
#include <fast_io_device.h>
#include <random>

int main() {
    using namespace ::fast_io::io;

    // Create a cryptographically secure random engine
    // Automatically uses the best platform-specific entropy source
    ::fast_io::ibuf_white_hole_engine eng;

    // Use uniform_int_distribution for uniform randomness in a range
    ::std::uniform_int_distribution<::std::uint32_t> dist(0, 99);

    // Generate random numbers in [0, 99] with perfect uniform distribution
    for (::std::size_t i{}; i != 10; ++i) {
        println("Random number in [0, 99]: ", dist(eng));
    }
}

The distribution calls eng() to get raw random bits, then maps them to the desired range using rejection sampling, ensuring each value in [0, 99] has exactly the same probability of being selected.

Key Takeaways

  • Never use modulo operations (like value % range) to generate random numbers in a range.
  • Modulo creates biased distributions due to the pigeonhole principle — some values will be slightly more likely than others.
  • Always use ::std::uniform_int_distribution or other distributions from <random>.
  • Distribution engines use rejection sampling to ensure perfect uniformity.
