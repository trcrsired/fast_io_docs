Ch12.1: Overview and Key Concepts

Why Use fast_io's Random Engine?

C++'s <random> library provides powerful tools, but using them correctly requires care:

  • Manual seeding is error-prone: You must seed generators like ::std::mt19937 with a truly random seed, or all outputs are predictable. Using ::std::time(nullptr) is insecure and low-entropy.
  • Modulo bias: Converting random values to a range using modulo (value % range) creates biased distributions due to the pigeonhole principle.
  • Platform selection: You need to know which OS-specific API to call (getrandom(), /dev/urandom, RtlGenRandom, etc.).

fast_io's ibuf_white_hole_engine solves all these problems by providing a cryptographically secure, buffered engine that automatically uses the best platform-specific entropy source and works directly with C++ distributions.

Platform-Specific Random Sources

fast_io provides several random number sources, each optimized for its platform:

Type Platform Source
linux_getrandom Linux getrandom() system call
posix_dev_urandom POSIX systems /dev/urandom device
bsd_arc4random BSD systems arc4random() function
win32_crypt_gen_random Windows CryptGenRandom() API
wasi_random_get WASI (WebAssembly) WASI random API

All these sources implement the input stream concept, meaning you can read random bytes from them using the same API as any other input stream.

Reading Raw Random Bytes

You can read raw random bytes from these sources using read_all or other input operations. This is useful when you need random data for seeds, tokens, or other purposes. 


#include <fast_io.h>
#include <fast_io_device.h>

int main() {
    using namespace ::fast_io::io;

    // Create a random source (platform-specific)
#if defined(__linux__)
    ::fast_io::linux_getrandom rng;
#elif defined(_WIN32)
    ::fast_io::win32_crypt_gen_random rng;
#else
    ::fast_io::posix_dev_urandom rng;
#endif

    // Read 16 random bytes
    ::std::byte buffer[16];
    ::fast_io::io::read_all(rng, buffer, buffer + 16);

    // Print the random bytes
    println("Random bytes:");
    for (::std::size_t i{}; i != 16; ++i) {
        print(::fast_io::mnp::hex(::fast_io::mnp::upper(buffer[i])), " ");
    }
    println();
}

However, for most use cases, you should use ibuf_white_hole_engine with distributions instead of reading raw bytes directly.
