Ch9.8: vector Capacity Management

Overview

Understanding how ::fast_io::vector manages its capacity is essential for writing efficient code. A vector maintains two quantities:

  • size: the number of elements currently stored.
  • capacity: the number of elements the vector can hold before it must allocate more memory.

#include <fast_io_dsal/vector.h>
#include <fast_io.h>

int main() {
    using namespace ::fast_io::iomnp;

    ::fast_io::vector<::std::size_t> v;
    println("Initial: size=", v.size(), " capacity=", v.capacity());

    v.push_back(1);
    println("After 1 push: size=", v.size(), " capacity=", v.capacity());
}

1. reserve()

reserve(n) ensures that the vector has enough capacity for at least n elements. It does not change the size. If the current capacity is already ≥ n, this is a no-op. 


::fast_io::vector<::std::size_t> v;
v.reserve(1000zu); // Allocate space for 1000 elements upfront
// size() == 0, capacity() >= 1000

2. shrink_to_fit()

shrink_to_fit() requests that the vector reduce its capacity to match its size. This can reclaim memory after erasing many elements. 


::fast_io::vector<::std::size_t> v;
v.reserve(10000zu);
for (::std::size_t i{}; i < 5zu; ++i) v.push_back(i);
// size() == 5, capacity() >= 10000

v.shrink_to_fit();
// size() == 5, capacity() == 5

3. push_back_unchecked()

When you have pre-allocated capacity with reserve(), you can use push_back_unchecked() to avoid the capacity check on each insertion: 


::fast_io::vector<::std::size_t> v;
v.reserve(100zu);
for (::std::size_t i{}; i < 100zu; ++i) {
    v.push_back_unchecked(i); // No capacity check - fast!
}
// size() == 100, capacity() >= 100

4. clear() vs clear_destroy()

These two operations differ in what happens to the allocated memory:

Operation Elements Memory Use case
clear() Destroyed Retained (capacity unchanged) You plan to reuse the vector
clear_destroy() Destroyed Deallocated (capacity becomes 0) You want to free memory 

::fast_io::vector<::std::size_t> v{1zu, 2zu, 3zu};
v.reserve(1000zu);

v.clear();
// size() == 0, capacity() >= 1000 (memory still held)

v.clear_destroy();
// size() == 0, capacity() == 0 (memory freed)

Key takeaways

  • capacity() is the allocated space; size() is the used space.
  • reserve(n) pre-allocates but does not change size.
  • shrink_to_fit() reduces capacity to match size.
  • reserve() + push_back_unchecked() is the fastest way to fill a vector.
  • clear() keeps memory; clear_destroy() frees it.
